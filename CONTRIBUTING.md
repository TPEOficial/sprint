# Contributing to Sprint

Thank you for your interest in contributing to Sprint 🚀

Sprint is an experimental next-generation edge server framework focused on performance, structure, and developer experience.

---

## 📌 Code Style

All contributions MUST follow the **Ciphera Style** guidelines.

The official style guide is available here:

https://docs.tpeoficial.com/docs/ciphera/ciphera-style/getting-started

Pull requests that do not follow the Ciphera Style may be requested to update formatting before review.

---

## 🛠 Development Workflow

1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Write clean, well-documented code.
4. Ensure there are no lint or build errors.
5. Submit a Pull Request with a clear description.

---

## 🧪 Testing

Before submitting changes:

- Ensure the project builds successfully.
- Verify that existing functionality is not broken.
- If adding new features, include appropriate tests where applicable.

---

## 🔒 Security

Do not introduce:
- Breaking security changes.
- Hardcoded secrets.
- Unsafe defaults.

Security improvements are welcome and encouraged.

If you discover a vulnerability, please report it privately following the `SECURITY.md` policy.

---

## 📦 Commit Guidelines

- Use clear and descriptive commit messages.
- Keep commits focused on a single change.
- Follow conventional and clean commit structure.

---

## 🙏 Code of Conduct

By contributing, you agree to follow the project's Code of Conduct.

Thank you for helping improve Sprint.
